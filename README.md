# CS305-Software-Security
Portfolio artifacts and reflections from CS 305 Software Security.
Artemis Financial is a financial services company that needed to improve the security of its software application used to protect sensitive customer information. The company wanted to identify vulnerabilities within its application and implement secure coding practices to reduce security risks. During the vulnerability assessment, I successfully identified potential weaknesses and implemented security improvements, including enabling HTTPS communication and using cryptographic hashing to ensure data integrity. Coding securely is important because it helps protect confidential data, reduces the risk of cyberattacks, and strengthens customer trust while supporting the organization's overall stability and reputation.

One of the most helpful aspects of this project was using OWASP Dependency-Check to identify known vulnerabilities in project dependencies. Interpreting the vulnerability reports and distinguishing between legitimate threats and false positives was challenging, but it provided valuable experience in vulnerability analysis. To increase layers of security, I implemented SSL certificates, performed dependency scans, and followed secure coding practices throughout development. In future projects, I would continue using tools such as OWASP Dependency-Check, static code analysis tools, and industry security standards to assess vulnerabilities and determine appropriate mitigation strategies.

To ensure that the software remained functional and secure, I tested the application after each modification and verified that it built and executed successfully without errors. After refactoring the code, I reran dependency scans and tested application functionality to confirm that no new vulnerabilities had been introduced. The resources and tools used throughout this project, including Maven, Spring Boot, OWASP Dependency-Check, and Java security utilities, will be useful in future coursework and professional development. For future employers, I would showcase the completed vulnerability assessment and secure software reports as evidence of my ability to analyze security risks, implement secure coding practices, and maintain secure software applications.
